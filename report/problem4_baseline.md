## 1. 问题目标

第四题的核心任务不是预测，而是建立一个水质风险评价体系。评价对象为出厂水浊度：

```text
NTU
```

约束标准为：

```text
NTU ≤ 1
```

因此第四题的建模目标可以表述为：

```text
以出厂水浊度 NTU 为核心指标，结合超标幅度、超标持续时长和连续超标时长，构建日尺度水质风险评价体系，并将 2026 年近 3 个月水质状态划分为安全、低风险、中风险和高风险。
```

## 3. 运行日定义

由于水厂数据每 2 小时记录一次，且运行周期跨自然日，baseline 采用运行日 `OP_DATE` 而不是自然日直接统计。

运行日定义为：

```text
当天 07:00 至次日 05:00 归为同一个 OP_DATE
```

即：

```text
07:00, 09:00, 11:00, 13:00, 15:00, 17:00, 19:00, 21:00, 23:00,
次日 01:00, 03:00, 05:00
```

共 12 个时间点，构成一个完整运行日。

这样处理的好处是：

```text
1. 与水厂实际运行班次更一致；
2. 避免凌晨 01:00、03:00、05:00 被错误归入下一自然日；
3. 每个运行日形成固定 12 个观测点，便于计算超标持续时长。
```

## 4. 风险评价指标体系

第四题 baseline 先对每个时间点计算是否超标，再聚合到每日尺度。

### 4.1 点位级超标状态

对每一个时间点 \(t\)，定义：

\[
I_t =
\begin{cases}
1, & NTU_t > 1 \\
0, & NTU_t \leq 1
\end{cases}
\]

其中：

```text
I_t = 1 表示该时间点超标；
I_t = 0 表示该时间点正常。
```

超标幅度定义为：

\[
E_t = \max(NTU_t - 1, 0)
\]

其中：

```text
E_t = 0 表示未超标；
E_t > 0 表示超过 1 NTU 的幅度。
```

---

### 4.2 日尺度风险指标

对每个运行日 \(d\)，计算以下指标：

| 指标 | 公式 / 定义 | 含义 |
|---|---|---|
| `n_records` | 当日有效记录数 | 当日 NTU 有效观测数 |
| `mean_NTU` | \(\frac{1}{n}\sum NTU_t\) | 当日平均浊度 |
| `max_NTU` | \(\max(NTU_t)\) | 当日最大浊度 |
| `min_NTU` | \(\min(NTU_t)\) | 当日最小浊度 |
| `std_NTU` | `std(NTU_t)` | 当日浊度波动程度 |
| `exceed_count` | \(\sum I_t\) | 当日超标时间点数量 |
| `exceed_ratio` | \(\frac{\sum I_t}{n}\) | 超标比例 |
| `exceed_duration_hours` | `exceed_count × 2` | 累计超标持续时长 |
| `max_excess` | \(\max(E_t)\) | 最大超标幅度 |
| `mean_excess` | \(\frac{1}{n}\sum E_t\) | 平均超标幅度 |
| `total_excess` | \(\sum E_t\) | 累计超标幅度 |
| `excess_area` | `total_excess × 2` | 超标面积，近似反映超标强度与持续时间 |
| `max_consecutive_exceed_hours` | 最长连续超标点数 × 2 | 最长连续超标持续时长 |

其中，数据间隔为 2 小时，因此：

```text
1 个超标点 = 2 小时超标持续时间
```

## 5. 风险等级划分规则

baseline 将每日风险划分为四个等级：

```text
安全
低风险
中风险
高风险
```

具体规则如下。

| 风险等级 | 英文标记 | 判定规则 | 含义 |
|---|---|---|---|
| 安全 | `SAFE` | `max_NTU ≤ 1` | 全天无超标 |
| 低风险 | `LOW_RISK` | `max_NTU > 1` 且 `max_excess ≤ 0.2` 且 `exceed_duration_hours ≤ 2` | 轻微短时超标 |
| 中风险 | `MEDIUM_RISK` | 有超标，但不属于低风险，也未达到高风险 | 存在一定超标风险 |
| 高风险 | `HIGH_RISK` | `max_excess > 0.5` 或 `exceed_duration_hours ≥ 6` 或 `max_consecutive_exceed_hours ≥ 4` | 大幅度或长时间持续超标 |

其中风险阈值为：

| 参数 | 数值 | 含义 |
|---|---:|---|
| `NTU_THRESHOLD` | 1.0 | 出厂水浊度限值 |
| `LOW_RISK_MAX_EXCESS` | 0.2 | 低风险最大超标幅度 |
| `LOW_RISK_MAX_DURATION_HOURS` | 2.0 | 低风险最大超标持续时长 |
| `HIGH_RISK_MAX_EXCESS` | 0.5 | 高风险最大超标幅度阈值 |
| `HIGH_RISK_MIN_DURATION_HOURS` | 6.0 | 高风险累计超标持续时长阈值 |
| `HIGH_RISK_MIN_CONSECUTIVE_HOURS` | 4.0 | 高风险连续超标时长阈值 |

该规则同时考虑了：

```text
1. 是否超标；
2. 超标幅度；
3. 累计超标持续时间；
4. 连续超标持续时间。
```

因此比单纯用 `max_NTU > 1` 判断风险更稳健。

## 12. 论文中需要注意的点

### 12.1 不要把第四题写成预测模型

第四题当前 baseline 是：

```text
rule-based risk assessment
```

不是：

```text
RF / XGBoost / GRU prediction model
```

因此论文中应强调：

```text
评价体系
风险指标
风险等级划分规则
```

而不是模型训练精度。

---

### 12.2 需要解释为什么使用最大值、时长和连续时长

单纯判断 `max_NTU > 1` 不够，因为一次短暂轻微超标和长时间严重超标的风险程度不同。

因此 baseline 同时引入：

```text
最大超标幅度
累计超标持续时长
最长连续超标时长
```

这样可以区分：

```text
短时轻微异常 → 低风险
持续或大幅度异常 → 高风险
```

---

### 12.3 有效评价日数问题

当前结果中：

```text
2026-01-01 至 2026-03-31 的有效评价日数为 62 天
```

而不是完整自然日 90 天。原因是 notebook 在进入风险评价前执行了：

```python
risk_df = risk_df.dropna(subset=["NTU"])
```

因此只有存在有效 `NTU` 记录的运行日才参与统计。

论文中建议写成：

```text
在剔除 NTU 缺失记录后，共得到 62 个有效运行日用于风险评价。
```

如果后续补齐 2 月或其他缺失数据，需要重新运行 notebook 并更新结果。
