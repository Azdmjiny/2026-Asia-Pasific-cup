# Problem 1 NTU 自相关性与周期性分析报告

## 1. 分析目的

前面的滞后相关性分析主要研究其他水质变量和工艺变量与当前出厂水浊度 `NTU` 之间的关系，即：

$$
corr(x_{t-k}, NTU_t)
$$

为了进一步判断出厂水浊度 `NTU` 自身是否具有时间连续性和周期性，本阶段补充进行 `NTU` 自相关性分析和长滞后周期性分析。

本阶段主要回答以下问题：

1. 当前 `NTU` 是否受到前几个时刻 `NTU` 的影响；
2. `NTU` 是否具有明显短期惯性；
3. `NTU` 是否在 24 小时、48 小时、72 小时等位置存在周期性重复；
4. `NTU_lag` 是否适合作为后续主模型输入变量；
5. 周期性分析能否为后续模型加入时间特征提供依据。

由于原始数据每 2 小时记录一次，因此滞后阶数与时间的对应关系如下：

| 滞后阶数 | 实际含义 |
|---:|---|
| `lag1` | 2 小时前 |
| `lag2` | 4 小时前 |
| `lag3` | 6 小时前 |
| `lag6` | 12 小时前 |
| `lag12` | 24 小时前 |
| `lag24` | 48 小时前 |
| `lag84` | 168 小时前，即 7 天前 |

## 3. NTU 自相关性分析方法

NTU 自相关性分析用于判断出厂水浊度自身是否具有时间连续性。其基本形式为：

$$
corr(NTU_{t-k}, NTU_t), \quad k=0,1,\ldots,12
$$

其中：

- $NTU_t$ 表示当前时刻的出厂水浊度；
- $NTU_{t-k}$ 表示第 $k$ 阶滞后的出厂水浊度；
- `lag1` 表示 2 小时前；
- `lag12` 表示 24 小时前。

本阶段同时计算 Pearson 和 Spearman 相关系数：

| 指标 | 含义 |
|---|---|
| Pearson 相关系数 | 衡量线性自相关关系 |
| Spearman 相关系数 | 衡量单调自相关关系，对异常值和非线性关系更稳健 |

需要注意的是，`lag0` 表示当前 `NTU` 与自身比较，因此相关系数必然为 1，不用于判断真实时间延续性。实际分析重点为 `lag1` 至 `lag12`。

---

## 4. NTU 自相关性结果

短滞后自相关分析结果显示，`NTU` 自身存在明显的短期时间连续性。

其中，`lag1` 的自相关性最强：

| 滞后项 | 滞后时间 | Pearson 相关系数 | Spearman 相关系数 | 解释 |
|---|---:|---:|---:|---|
| `NTU_lag1` | 2 小时前 | 约 0.881 | 约 0.970 | 极强短期连续性 |
| `NTU_lag12` | 24 小时前 | 约 0.461 | 需结合实际输出 | 仍存在一定相关性 |

该结果说明：

1. 出厂水浊度在相邻时间点之间具有明显延续性；
2. 2 小时前的 `NTU` 对当前 `NTU` 具有很强的统计关联；
3. 即使滞后到 24 小时，`NTU` 与当前值仍可能保持一定相关性；
4. `NTU` 的变化不是完全随机的，而是具有明显时间惯性。

因此，从统计角度看，历史 `NTU` 对当前 `NTU` 具有较高预测价值。

---

## 5. NTU_lag 是否能直接作为主模型输入

虽然 `NTU_lag1`、`NTU_lag2` 等历史项具有很强相关性，但在本题中不能简单将其作为主模型核心输入。

原因是题目要求预测指定日期：

| 预测日期 |
|---|
| 2026-02-01 |
| 2026-02-10 |
| 2026-02-20 |

这些日期的 `NTU` 本身属于待预测目标。如果直接使用 `NTU_lag1`、`NTU_lag2` 作为输入，则在目标日期内部会出现递推依赖问题。

例如，预测 `2026-02-01 09:00` 时，如果需要 `NTU_lag1`，则它对应 `2026-02-01 07:00` 的 `NTU`。但 `2026-02-01 07:00` 的 `NTU` 也可能是缺失并需要模型预测的值。因此，模型会从“使用真实历史值”变成“使用前一步预测值”，形成递推预测。

这种方式并非不能做，但它属于递推时间序列模型，误差可能逐步累积。对于第一版主模型，更稳妥的做法是优先使用可直接观测的外部变量，例如：

| 推荐外部变量 |
|---|
| `FILT. NTU` |
| `CLR` |
| `R/W FLOW` |
| `T/W FLOW` |
| `C/W WELL LEVEL` |
| `R/W NTU` |
| `ALUM` |
| `CL2` |

因此，本阶段对 `NTU_lag` 的分析主要用于说明 `NTU` 具有时间连续性，而不建议直接把 `NTU_lag` 作为主模型第一版核心输入。

## 6. 运行日 OP_DATE 的必要性

在检查指定三天的 `NTU_lag` 可用性时，需要注意水厂数据的日期结构。

原始数据的一天不是普通自然日的 00:00 至 23:59，而更接近运行日结构：

| 运行日时间点 |
|---|
| 07:00 |
| 09:00 |
| 11:00 |
| 13:00 |
| 15:00 |
| 17:00 |
| 19:00 |
| 21:00 |
| 23:00 |
| 次日 01:00 |
| 次日 03:00 |
| 次日 05:00 |

因此，不能简单使用自然日期 `DATE_ONLY` 筛选目标日期。否则，某些 01:00、03:00、05:00 数据会被错误归入当天，而实际上它们属于前一天的运行日。

更合理的处理方式是构造运行日期 `OP_DATE`：

| 规则 | 解释 |
|---|---|
| 若时间早于 07:00 | 归属于前一天运行日 |
| 若时间为 07:00 及以后 | 归属于当天运行日 |

例如：

| 实际时间 | 运行日 |
|---|---|
| 2026-02-02 01:00 | 2026-02-01 |
| 2026-02-02 03:00 | 2026-02-01 |
| 2026-02-02 05:00 | 2026-02-01 |

使用 `OP_DATE` 后，才能正确筛选题目指定的三个预测运行日。

## 7. 周期性分析方法

为了判断 `NTU` 是否存在周期性变化，本阶段将最大滞后阶数从 `lag12` 扩展到 `lag84`。

由于数据每 2 小时记录一次：

$$
lag84 = 84 \times 2h = 168h = 7days
$$

因此，长滞后自相关分析形式为：

$$
corr(NTU_{t-k}, NTU_t), \quad k=0,1,\ldots,84
$$

本阶段重点检查以下典型周期位置：

| 滞后项 | 对应时间 | 含义 |
|---:|---:|---|
| `lag12` | 24 小时 | 1 天周期 |
| `lag24` | 48 小时 | 2 天周期 |
| `lag36` | 72 小时 | 3 天周期 |
| `lag48` | 96 小时 | 4 天周期 |
| `lag60` | 120 小时 | 5 天周期 |
| `lag72` | 144 小时 | 6 天周期 |
| `lag84` | 168 小时 | 7 天周期 |

如果 `NTU` 存在明显日周期，则自相关曲线应在 `lag12`、`lag24`、`lag36` 等 24 小时倍数位置附近反复出现局部峰值。

如果自相关系数只是随着滞后时间逐渐下降，而没有在这些位置形成明显局部峰值，则说明 `NTU` 更主要表现为短期时间惯性，而不是稳定周期性。

---

## 8. 周期性判断标准

本阶段不只看某个周期位置的相关系数大小，还进一步判断该周期位置是否为局部峰值。

对于某一周期滞后项，例如 `lag12`，如果满足：

$$
corr(lag12) > corr(lag11)
$$

且：

$$
corr(lag12) > corr(lag13)
$$

则认为 `lag12` 是自相关曲线中的局部峰值。

判断逻辑如下：

| 情况 | 解释 |
|---|---|
| 短滞后相关性很高，长滞后逐渐下降 | 主要是时间惯性 |
| `lag12`、`lag24`、`lag36` 附近反复出现局部峰值 | 存在较明显日周期 |
| 周期位置相关性不高，且不是局部峰值 | 周期性证据不足 |
| `lag84` 相关性较高 | 可能存在周周期，但需谨慎解释 |

---

## 9. 周期性分析结果解释

根据长滞后自相关分析，可以从两个方面理解 `NTU` 的动态特征。

### 9.1 短期惯性

短滞后结果显示，`NTU_lag1` 与当前 `NTU` 的相关性很高，说明出厂水浊度具有明显短期惯性。也就是说，当前出厂水浊度往往受到前一个或前几个时间点状态的影响。

这与自来水处理过程一致。出厂水来自前端处理过程和清水池调蓄，水质变化通常不会在一个采样间隔内完全突变，而是具有一定连续性。

### 9.2 日周期可能性

若长滞后自相关曲线在 `lag12`、`lag24`、`lag36` 等位置仍保持较高相关性，并且这些位置是局部峰值，则可以认为 `NTU` 存在一定日周期结构。

这种周期性可能来自：

1. 日内用水量变化；
2. 水厂运行调度；
3. 原水流量变化；
4. 清水池水位变化；
5. 加药和过滤运行节律；
6. 水力停留时间的日内变化。

但如果这些 24 小时倍数位置不是局部峰值，而只是随滞后时间逐渐下降，则说明 `NTU` 的主要特征仍然是时间惯性，而不是强周期性。

因此，周期性结论需要结合 `ntu_periodicity_daily_peak_check.csv` 和 `ntu_periodicity_summary.csv` 中的结果进行判断。

## 10. 对后续建模的影响

本阶段结果对后续建模有三点启示。

### 10.1 NTU 历史值具有预测价值，但不适合作为第一版主模型核心输入

`NTU_lag1` 与当前 `NTU` 高度相关，说明历史出厂水浊度确实有预测价值。但由于指定预测日期中 `NTU` 本身缺失，直接使用 `NTU_lag` 会引入递推预测问题。

因此，第一版主模型仍建议优先使用可直接观测的外部变量：

| 主模型优先变量 |
|---|
| `FILT. NTU` |
| `CLR` |
| `R/W FLOW` |
| `T/W FLOW` |
| `C/W WELL LEVEL` |
| `R/W NTU` |

### 10.2 可考虑加入时间周期特征

如果周期性分析显示 `NTU` 在 24 小时倍数位置存在一定重复结构，则后续模型可以加入时间特征，而不是直接使用 `NTU_lag`。

可选时间特征包括：

| 特征 | 含义 |
|---|---|
| `hour` | 当前小时 |
| `hour_sin` | 日内周期的正弦编码 |
| `hour_cos` | 日内周期的余弦编码 |
| `dayofweek` | 星期几 |
| `dayofweek_sin` | 周期化星期特征 |
| `dayofweek_cos` | 周期化星期特征 |

其中，`hour_sin` 和 `hour_cos` 比直接使用 `hour` 更适合表达周期性，因为它们可以反映 23:00 与 01:00 在时间上接近。

### 10.3 可作为扩展模型尝试递推预测

如果时间允许，可以构造扩展模型：

$$
NTU_t = f(NTU_{t-1}, NTU_{t-2}, X_t)
$$

但该模型在预测目标日期时需要递推：

1. 先预测第一个时间点；
2. 再将第一个预测值作为后续时间点的 `NTU_lag1`；
3. 依次预测当天后续时间点。

这种模型可能提升短期预测能力，但也可能造成误差累积。因此，更适合作为扩展对比模型，而不适合作为第一版主模型。

## 12. 可供论文正文使用的总结表述

通过对出厂水浊度 `NTU` 进行自相关性分析，发现 `NTU` 在短滞后范围内具有明显时间连续性。其中，`NTU_lag1` 与当前 `NTU` 的相关性最高，说明 2 小时前的出厂水浊度与当前出厂水浊度高度相关，出厂水浊度序列表现出较强短期惯性。

进一步将自相关分析扩展到 7 天范围，用于判断 `NTU` 是否存在周期性变化。若自相关曲线在 24 小时及其倍数位置附近出现局部峰值，则说明 `NTU` 可能具有日周期结构；若曲线主要表现为随滞后时间增加逐渐衰减，则说明 `NTU` 的动态变化主要由短期惯性主导，而不是显著周期性主导。

需要注意的是，虽然历史 `NTU` 对当前 `NTU` 具有较强统计关联，但由于题目指定预测日期中的 `NTU` 本身为待预测变量，直接使用 `NTU_lag` 作为模型输入会引入递推预测问题。因此，本文将 `NTU` 自相关性和周期性分析主要作为动态特征分析依据，而主预测模型仍优先使用可直接观测的外部变量，如 `FILT. NTU`、`CLR`、`R/W FLOW`、`T/W FLOW` 和 `C/W WELL LEVEL`。

## 13. 给论文手的注意事项

论文中可以写：

- `NTU` 具有明显短期时间连续性；
- `NTU_lag1` 与当前 `NTU` 高度相关；
- 周期性需要结合 24 小时倍数位置是否形成局部峰值判断；
- 自相关结果说明 `NTU` 序列具有动态结构；
- 但主模型不直接使用预测日期中不可获得的 `NTU_lag`。

论文中不建议写：

- 直接使用 `NTU_lag1` 作为最终主模型核心变量；
- 只因为 `lag12` 相关性不低，就断言存在严格日周期；
- 忽略运行日 `OP_DATE`，直接用自然日筛选目标日期；
- 把自相关解释为因果关系。

更稳妥的表述是：

数学上的自相关性说明 `NTU` 具有时间连续性和潜在周期结构，但最终模型特征选择仍需考虑目标日期变量可获得性、递推预测风险和模型稳定性。
